Extensions API Reference

Header: extensions/Include/Extensions.h

Interface for customizing URM behavior at build time: registering custom resource applier/teardown callbacks, overriding configuration YAML files, and registering post-processing callbacks.

Note

Extension macros must be used in the global scope of a translation unit. They rely on static initialization to register callbacks before main().

URM_REGISTER_RES_APPLIER_CB(resCode, resourceApplierCallback)

Register a Customer Resource Applier Callback for a particular ResCode.

Note

This macro must be used in the Global Scope.

Parameters:

  • resCode – An unsigned 32-bit integer representing the Resource ResCode.

  • resourceApplierCallback – A function Pointer to the Custom Applier Callback.

URM_REGISTER_RES_TEAR_CB(resCode, resourceTearCallback)

Register a Customer Resource Teardown Callback for a particular ResCode.

Note

This macro must be used in the Global Scope.

Parameters:

  • resCode – An unsigned 32-bit integer representing the Resource ResCode.

  • resourceTearCallback – A function Pointer to the Custom Teardown Callback.

URM_REGISTER_CONFIG(configType, yamlFile)

Register custom Config (YAML) file. This Macro can be used to register Resource Configs File, Signal Configs file and others with Resource Tuner.

Note

This macro must be used in the Global Scope.

Parameters:

  • configType – The type of Config for which the Custom YAML file has to be specified.

  • yamlFile – File Path of this Config YAML file.

URM_REGISTER_POST_PROCESS_CB(identifier, callback)

Register post processing callbacks for different workloads and per-process.

Note

This macro must be used in the Global Scope.

Parameters:

  • identifier – The key (string) identifier for the process or workload

  • callback – The post processing Callback to be registered

Extensions.

Provides an Interface for Customizing Resource Tuner Behaviour. Through the Extension Interface, Custom Resource Callbacks / Appliers as well as Custom Config Files (for example: Resource Configs or Signal Configs) can be specified.
